1. 상속관계의 클래스를 작성하기 위해서는 각각의 클래스에 대한 설계가 먼저 이루어져야 한다.
2. 클래스의 설계는 현실 객체를 소프트웨어 객체로 변환하는 과정이다.
    현실 객체를 SW객체로 정의하기 위해서는 프로그램에 꼭 필요한 특징만 남기고 나머지 특징은 버린다.(추상화)
    객체는 속성과 행위로 이루어지는데 속성은 정적인 특징으로 상태라고도 하고 행위는 동적인 특징으로 동작이라고도 한다.
    객체의 속성은 클래스의 필드로, 객체의 행위는 클래스의 메소드로 설계된다.

개발자 속성: 이름, 나이, 성별, 개발언어
    행위: 먹는다, 잔다, 운동한다, 일한다(개발한다)

디자이너 속성: 이름, 나이, 성별, 디자인도구
        행위: 먹는다, 잔다, 운동한다, 일한다(디자인한다)

직장인 공통속성: 이름, 나이, 성별
    공통행위: 먹는다, 잔다, 운동한다, 일한다(추상메소드)

Worker(부모클래스:추상클래스)
fields - name:String, age:int, gender:String
methods - eat():void, sleep():void, exercise():void, work():void

Developer(자식클래스)
fields - language:String
methods - work():void

Designer(자식클래스)
fields - tool:String
methods - work():void
